Introduction

        Sass is a CSS pre-processor.

        Sass reduces repetition of CSS and therefore saves time.
        
        What is Sass?
        
        Sass stands for Syntactically Awesome Stylesheet
        Sass is an extension to CSS
        Sass is a CSS pre-processor
        Sass is completely compatible with all versions of CSS
        Sass reduces repetition of CSS and therefore saves time
        Sass was designed by Hampton Catlin and developed by Natalie Weizenbaum in 2006
        Sass is free to download and use


        Why Use Sass?
        
        Stylesheets are getting larger, more complex, and harder to maintain. This is where a CSS pre-processor can help.
        
        Sass lets you use features that do not exist in CSS, like variables, nested rules, mixins, imports, inheritance,
        built-in functions, and other stuff.

        The benefit of SASS is for example you define a color as a variable and use 
        this variable wherever you want.

        Ionic uses SASS.
        The main core of Bootstrap is also based on SASS.

        The output of a SASS program is a css file.

        The next generation of SASS is SCSS and it is a little better.

Sass Variables


            Variables are a way to store information that you can re-use later.
            
            With Sass, you can store information in variables, like:
            
            strings, numbers, colors, booleans, lists, nulls
            
            Sass uses the $ symbol, followed by a name, to declare variables:
            
            $variable_name: value;

SASS Variable Scope

        We define sass variables at the beginning of the scss file.
        This way those variables are known to all the {} spaces inside the file.

        If a variables is defined inside one of the {} spaces it is only known there 
        unless use !global after it like:
        {$myColor: green !global}

SASS Mixin

        When we want to pack some css code and avoid retyping the we use mixins.
        Mixins are like variables but they are more strong.

        To define a mixin use @ sigh like: @mixin mixin_name{}
        To use it inside sass file use bellow structure:
        @include mixin_name();

        Include a mixin on the top of all the other properties
        in order to you be able to overwrite them.

        Mixin_Example_1.html
        Mixin_Example_1.scss
        Mixin_Example_1.css
        
        Tip: Hyphens and underscores are considered to be the same. This means that
        @mixin important-text { } and @mixin important_text { } are considered as the same mixin!

        A mixin can also include other mixins:
        
        @mixin special-text {
        @include important-text;
        @include link;
        @include special-border;
        }
